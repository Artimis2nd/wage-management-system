ปัญหาที่อัพโหลดรูปไม่สำเร็จแล้วบันทึกไม่ได้

กระบวนการแก้ไขทั้งหมด เราเจอปัญหาหลักๆ 2 ด่าน ที่ต้องผ่านไปให้ได้ครับ:

ด่านที่ 1: ปัญหา "ใบอนุญาต" (The Permission Problem)
อาการ: API Error: Exception: ไม่ได้รับอนุญาตให้เข้าถึง: DriveApp
สาเหตุ: ปัญหานี้เปรียบเสมือน "ยามหน้าประตู" ที่ไม่ยอมให้สคริปต์ของเราเข้าไปใน "โกดัง" (Google Drive) เลยแม้แต่น้อย ไม่ว่าโค้ดข้างในจะเขียนดีแค่ไหนก็ตาม
วิธีแก้: คือทั้งหมดที่เราทำกันเรื่อง "สิทธิ์" ครับ
การล้างสิทธิ์เก่าในบัญชี Google (Revoke Access)
การกด Run เพื่อขอสิทธิ์ใหม่
การสร้าง New deployment พร้อมตั้งค่า Execute as: Me และ Who has access: Anyone
ผลลัพธ์: เมื่อผ่านด่านนี้ได้ "ยาม" ก็จะยอมเปิดประตูให้สคริปต์ของเราเข้าไปทำงานใน Google Drive ได้แล้ว
ด่านที่ 2: ปัญหา "ความทนทานและการหมดเวลา" (The Robustness & Timeout Problem)
อาการ: หลังจากผ่านด่านแรกมาได้แล้ว เราเจอปัญหาใหม่คือ "รูปอัปโหลดได้แค่รูปเดียว แล้วข้อมูลในชีตก็ไม่บันทึก"
สาเหตุ: ปัญหานี้เกิดจากโค้ด saveImagesToDrive เวอร์ชันเก่าที่ใช้ .map ครับ การทำงานของมันเปรียบเสมือน "สายพานการผลิตที่ถ้ามีของชิ้นเดียวพัง สายพานจะหยุดทำงานทั้งหมด"
เมื่อเราส่งรูปไป 3 รูป สคริปต์จะเริ่มบันทึกรูปแรก... รูปที่สอง...
แต่การบันทึกไฟล์ขนาดใหญ่ใช้เวลานาน พอถึงรูปที่สาม สคริปต์อาจจะทำงานเกินเวลาที่ Google กำหนด (Timeout) หรือเกิดข้อผิดพลาดอื่นๆ
เมื่อเกิดข้อผิดพลาดที่รูปที่สาม การทำงานของ .map จะ "พัง" และหยุดการทำงานทั้งหมดทันที ทำให้มันไม่ไปถึงขั้นตอนการบันทึกข้อมูลลง Google Sheet
วิธีแก้: คือสิ่งที่คุณชี้ให้เห็นอย่างถูกต้องครับ! เราเปลี่ยนมาใช้ .forEach ที่มี try...catch อยู่ข้างใน
การทำงานแบบใหม่นี้เปรียบเสมือน "พนักงานที่ทำงานทีละชิ้น"
พนักงานจะหยิบรูปแรกไปบันทึก -> สำเร็จ
หยิบรูปที่สองไปบันทึก -> สำเร็จ
หยิบรูปที่สามไปบันทึก -> สมมติว่าเกิดข้อผิดพลาด -> try...catch จะเข้ามาจัดการ โดยแค่บันทึกว่า "งานชิ้นนี้พังนะ" แต่พนักงานจะไม่หยุดทำงาน
พนักงานจะทำงานในส่วนของตัวเองจนครบ แล้วเดินไปทำขั้นตอนต่อไป คือ "บันทึกข้อมูลลง Google Sheet" ได้สำเร็จ
